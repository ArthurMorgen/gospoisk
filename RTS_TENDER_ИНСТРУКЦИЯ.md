### Шаг 2: Если сайт заблокировал запрос

**Признаки:**
- `debug_rts_tender.html` пустой
- В нем captcha или защита
- Ошибка 403/503

**РЕШЕНИЕ:**

1. Откройте https://www.rts-tender.ru/poisk/poisk-44-fz/ в **браузере**
2. Дождитесь загрузки списка тендеров
3. Нажмите `Ctrl+S` → Сохраните страницу
4. Переместите файл в папку проекта как `debug_rts_tender.html`

---

### Шаг 3: Найти API (альтернативный путь)

Если сайт использует API (как zakupki.mos.ru):

1. Откройте https://www.rts-tender.ru/poisk/poisk-44-fz/
2. Нажмите `F12` → вкладка **Network**
3. Обновите страницу (`Ctrl+R`)
4. Найдите запросы типа:
   - `api/search`
   - `api/tenders`
   - `ajax/getTenders`
   - JSON/XML ответы

**ЕСЛИ НАШЛИ API:**
- Скопируйте URL запроса
- Скопируйте параметры (query params)
- Скопируйте пример ответа (JSON)
- Отправьте мне эти данные

---

### Шаг 4: Анализ HTML

После получения `debug_rts_tender.html`:

1. Откройте файл в браузере
2. Убедитесь что видны **карточки тендеров**
3. Нажмите `F12` → выберите элемент карточки тендера
4. Найдите:
   - **Класс контейнера карточки** (например: `div.tender-card`)
   - **Селектор названия** (например: `h3.title`)
   - **Селектор цены** (например: `.price`)
   - **Селектор ссылки** (например: `a.tender-link`)
   - **Селектор дедлайна** (например: `.deadline`)

## ❓ ЧАСТЫЕ ПРОБЛЕМЫ:

### Проблема: Сайт требует авторизацию

**Решение:** 
- Если тендеры видны только после входа → нужны cookies
- Авторизуйтесь в браузере
- Экспортируйте cookies (расширение "EditThisCookie")
- Добавим их в парсер

### Проблема: JavaScript динамическая загрузка

**Решение:**
- Используем Selenium (как планировали для zakupki.mos.ru)
- Медленнее, но работает

### Проблема: Captcha

**Решение:**
- Ищем API endpoint (обходит captcha)
- Или используем сервисы решения captcha
